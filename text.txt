✅ الخلاصة المختصرة أولاً (TL;DR)

    نوع المشروع: موقع ويب كامل لمقهى يسمى "Coffee Brand".

    الوظائف: عرض منتجات (قهوة/حلويات)، سلة مشتريات، نظام تسجيل دخول/خروج، نموذج تواصل، أسئلة شائعة، قسم خدمات، إحصائيات ديناميكية، وضع ليلي/نهاري.

    التقنيات المستخدمة:
    HTML5 CSS3 JavaScript (Vanilla) Supabase (PostgreSQL + Auth + Storage)

    هل هو ديناميكي؟ نعم، بشدة! البيانات تأتي من Supabase: المنتجات، الخدمات، خطط العضوية، الشهادات، الأسئلة الشائعة، ساعات العمل، الإحصائيات.
    فقط جزء صغير جداً (مثل بعض النصوص الثابتة والـ SVG) ثابت.

    آلية العمل:

        المستخدم يتفاعل مع الواجهة (ضغط على زر "أضف للسلة"، تسجيل دخول، إرسال نموذج).

        الـ JavaScript (وحدات منفصلة: auth.js, cart.js, products.js, services.js) يتواصل مع Supabase API.

        Supabase يقوم بمعالجة الطلبات (تسجيل/تسخير دخول، جلب المنتجات، إدخال رسائل التواصل، إلخ).

        يتم تحديث واجهة المستخدم ديناميكياً بدون إعادة تحميل الصفحة.

3️⃣ ملفات JavaScript (المنطق الديناميكي بالكامل)
js/config/supabase.js

    تهيئة اتصال Supabase (SUPABASE_URL, SUPABASE_ANON_KEY).

    يحتوي على كلاس SupabaseService مسئول عن:

        تسجيل دخول / إنشاء حساب / تسجيل خروج.

        عمليات CRUD عامة (getAll, insert, update, delete).

        رفع الصور إلى Storage.

        تحديث الواجهة تلقائياً عند تغير حالة المستخدم.

js/modules/auth.js

    يتحكم في نموذج تسجيل الدخول (sign-in.html) ونموذج إنشاء الحساب (sign-up.html).

    يتحقق من قوة كلمة المرور، صحة الإيميل، ويمنع التكرار (rate limiting).

    يعرض رسائل خطأ/نجاح منبثقة.

    عند نجاح تسجيل الدخول: يُظهر اسم المستخدم وزر Logout في كل الصفحات.

js/modules/cart.js

    يدير سلة المشتريات (تخزينها في localStorage).

    يحتوي على:

        إضافة/حذف/تحديث كمية منتج.

        فتح/إغلاق نافذة السلة المنزلقة.

        عرض السلة ديناميكياً.

        منع الوصول إلى الـ Checkout إذا كان المستخدم غير مسجل دخول (يظهر popup محمي).

js/modules/products.js

    يقوم بـ جلب المنتجات من Supabase وعرضها في menu.html.

    يدعم:

        فلترة (كل المشروبات / إسبريسو / بلوند روست / دارك روست / ديكاف).

        ترقيم صفحات (Pagination).

        نافذة عرض منتج منفصلة (تفاصيل، سعر، تغذية، مكونات، حساسية).

        تحميل صور كسول (Lazy loading).

        عرض منتجات مميزة في الصفحة الرئيسية.

js/modules/services.js

    المسئول عن صفحة الخدمات (services.html) بالكامل.

    يجلب ديناميكياً:

        مميزات الخدمات (services_features).

        خطط العضوية (membership_plans).

        معرض الصور (service_gallery).

        فوائد القهوة (coffee_benefits).

        باقات الخدمات (service_packages).

        شهادات العملاء (testimonials مع category = 'service').

    يدير نموذج الحجز (service_bookings) مع التحقق من تسجيل الدخول.

js/modules/contact.js

    يدير صفحة contact.html:

        نموذج الاتصال (contact_messages).

        النشرة البريدية (newsletter_subscribers).

        ساعات العمل (working_hours) وعرض "مفتوح/مغلق" حسب الوقت الفعلي.

        شهادات العملاء (testimonials).

        الإحصائيات (statistics).

js/modules/ui.js

    يحتوي على وظائف واجهة مستخدم مشتركة:

        وضع النهار/الليل (Light/Dark mode) مخزن في localStorage.

        زر العودة للأعلى.

        مؤشر تحميل (Loader) يختفي بعد تحميل الصفحة.

        أنيميشن التمرير (Scroll animations) للعناصر التي تحمل كلاس animate-left, animate-right, animate-up.

        أنيميشن الفوتر عند الوصول إليه.

        تحميل الصور بالـ Lazy loading باستخدام Intersection Observer.

        نافذة منبثقة مخصصة (showPopup).

        نافذة "يجب تسجيل الدخول" (showProtectedPopup).

js/modules/gallery.js

    يجلب صور معرض المنيو (gallery_images) ويعرضها في الفوتر (Footer).

js/main.js

    نقطة الدخول الرئيسية.

    ينتظر تحميل supabaseService ثم يقوم بتهيئة الموديولات حسب الصفحة الحالية.

    يدير القائمة الجانبية (Mobile Menu) مع طبقة تغطية (menu-overlay).

🔄 آلية العمل بالتفصيل (مثال على مسار معين)
مثال: مستخدم يضيف منتجاً إلى السلة ثم يضغط على Checkout

    المستخدم يفتح menu.html ← يتم استدعاء ProductsModule.init().

    ProductsModule يقوم بـ supabaseService.getAll('products') ويعرض المنتجات.

    المستخدم يضغط على زر "إضافة إلى السلة" (الزر يستمع للحدث عن طريق delegation).

    CartModule.addToCart(productId) يُستدعى → يخزن المنتج في localStorage ويعرض البادج (رقم 1) أعلى أيقونة السلة.

    المستخدم يضغط على أيقونة السلة → CartModule.openCart() يُظهر الـ Cart Sidebar مع المنتج.

    المستخدم يضغط على "Proceed to Checkout".

    CartModule.handleCheckout() يتحقق:

        إذا لم يسجل دخول → UIModule.showProtectedPopup() يُظهر نافذة "يجب تسجيل الدخول".

        إذا سجل دخول → يُظهر رسالة "Checkout feature coming soon".

مثال: تسجيل مستخدم جديد

    المستخدم يذهب إلى sign-up.html → AuthModule.init() يُهيئ النموذج.

    يملأ الإيميل، اسم المستخدم، كلمة المرور (يتم التحقق من قوتها فوراً).

    يضغط "Sign Up" → supabaseService.signUp(email, password, username).

    Supabase يُنشئ الحساب (إذا كان البريد غير موجود) ويرسل إيميل تأكيد إذا لزم الأمر.

    يظهر popup نجاح، وبعد 3 ثوانٍ يتم تحويله إلى sign-in.html.

    بعد تسجيل الدخول من sign-in.html، supabaseService يقوم بـ updateUIForAuthenticatedUser() في كل الصفحات (يُظهر اسم المستخدم وزر Logout، ويخفي أزرار Sign In/Sign Up).

باقي المشروع ديناميكي بالكامل.
📊 الأجزاء الرئيسية التي تعمل بكفاءة

    نظام المصادقة (Auth):

        تسجيل دخول/خروج/إنشاء حساب.

        عرض اسم المستخدم في الـ Header (جميع الصفحات).

        منع الوصول إلى الـ Checkout أو الحجز بدون تسجيل دخول.

    سلة المشتريات:

        إضافة/حذف/تحديث.

        حفظ السلة محلياً حتى لو أغلق المتصفح.

        نافذة منزلقة بتصميم جميل.

    صفحة المنيو:

        فلترة ديناميكية.

        ترقيم صفحات وتحميل سريع.

        نافذة عرض منتج تحتوي على معلومات غذائية ومكونات وحساسية.

    صفحة الخدمات:

        عرض خطط عضوية وباقات.

        نموذج حجز يتصل بـ Supabase مباشرة.

        إحصائيات وشهادات ديناميكية.

    صفحة الاتصال:

        ساعات عمل ديناميكية + حالة "مفتوح/مغلق" حسب الوقت الفعلي.

        نموذج اتصال يخزن الرسائل في Supabase.

        نموذج نشرة بريدية.

        شهادات عملاء ديناميكية.

    وضع الليل/النهار وتجربة المستخدم:

        زر واحد يغير الثيم بالكامل.

        أنيميشن سلس عند التمرير.

        زر العودة للأعلى.

        مؤشر تحميل (Loader) حتى تكتمل جميع العناصر.

⚠️ ملاحظات وأجزاء قد تحتاج إلى تطوير

    الدفع الإلكتروني (Checkout):
    حالياً مجرد رسالة "Coming soon". تحتاج إلى دمج بوابة دفع حقيقية.

    لوحة تحكم Admin:
    لا يوجد واجهة لإدارة المنتجات أو الخدمات حالياً.
    يجب الدخول إلى Supabase Dashboard مباشرة لإضافة/تعديل البيانات.

    البريد الإلكتروني:
    نموذج الاتصال يخزن في Supabase فقط (لا يرسل إيميل فعلياً).
    يمكن إضافة وظيفة إرسال إيميل باستخدام Supabase Edge Functions أو خدمة خارجية.

    تحسين محركات البحث (SEO):
    المحتوى الديناميكي قد لا يكون مرئياً لبعض محركات البحث.
    يُفضل إضافة Server-Side Rendering أو استخدام حل مثل @supabase/ssr.

    الأمان:
    سياسات Row Level Security (RLS) في Supabase يجب تكوينها لمنع المستخدمين من الوصول إلى بيانات بعضهم البعض.
